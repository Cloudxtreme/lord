Welcome to django-lord.

We are building an open source clone of a classic BBS door game that we've named "Lord of the Red Dragon".

Concept

We want to develop an engaging game platform that can be managed and extended by moderators and other developers. Just as the original game allowed modules to be built, our LORD game will be extendable via an API. We will also evaluate other platforms to plug into, such as facebook, ios and android apps.

The initial core game will resemble the original BBS door game to start, will grow as the community sees fit. The code repository can be forked from github, extended and then merged back into the trunk. The master branch will always be playable at the lordofthereddragon.com

How can you contribute?

Development

Want to get involved in programming? We need developers with Python and Django experience, as well as frontend html/css/js ninjas. To apply to be a regular contributor, simply choose an unassigned ticket from github, fork the LORD repository, finish the ticket and submit a pull request.

Design

We're always looking for design suggestions and original asset and artwork contributions.

Content

We need cartographers, doctor frankensteins and blacksmiths...

Testing

Test out the game at lordofthereddragon.com and submit tickets for bugs or enhancements on a href="https://github.com/django-lord/lord/issues?state=open">github.

Administration/Moderation

Help keep our world clean and running smoothly.

Development

Environment Setup

Clone the repository and setup the environment: 
$ git clone https://github.com/django-lord/lord.git
$ cd lord
$ virtuanenv venv
$ . venv/bin/activate
$ pip install -r requirements.txt
$ python manage.py syncdb
$ python manage.py migrate
$ python manage.py loaddata players/fixtures/*.json
$ python manage.py loaddata world/fixtures/*.json

Best Practices

All development is done in a fork or a milestone branch. Branches are named as "v" + milestone.name. milestone.name = target release date.

Hotfixes should be branched from master and named "hotfix-[issue.id]"

Design

For now we use twitter bootstrap since we don't have a frontend dev on the team yet.

Content

Maps: Design and create map layouts for new worlds. Suggestions for new map elements are welcome!

Monsters/Armor/Weapons: Creative new assets are needed as we expand levels.

Random Events: Walking through the forest is boring if nothing ever happens. We need to come up with ideas for random occurrences that happen as players explore our world.
